Step 2 — BackgroundFit Parameters
WPEM.BackgroundFit( intensity_csv, lowAngleRange, poly_n, bac_split, bac_num )
lowAngleRangeRequired
Low-Angle Cutoff (2θ °)
The minimum 2θ angle (in degrees) included in the background fit. Data below this threshold is excluded, which suppresses air-scatter artifacts and instrumental low-angle artefacts common in diffractometer setups. Increase this value if you observe a steep rising background near zero.
°
Range: 0 – 90° · Typical: 5 – 25°
poly_nRequired
Polynomial Degree
Degree of the polynomial used to model the background curve. Higher values allow the fitted background to follow complex curvatures (e.g., amorphous humps), but risk overfitting and spurious oscillations. For simple crystalline samples use 5–10; for amorphous-rich samples use 10–15.
Range: 1 – 30 · Typical: 8 – 15
bac_splitRequired
Background Anchor Segments
Number of equally spaced anchor regions (segments) used to define the piecewise background baseline. A larger value provides more local control over the background shape, useful when the background varies significantly across the angular range. Too many segments may over-segment the background.
Range: 2 – 100 · Typical: 8 – 30
bac_numOptional
Monte Carlo Optimization Steps
Number of stochastic optimization iterations used to fit the background polynomial. More iterations improve convergence quality but increase computation time linearly. For quick exploratory fits use 100–200; for publication-quality results use 500–1000+.
Range: 50 – 5000 · Typical: 200 – 600

CIF Preprocessing Parameters

        


          
          

            
filepathRequired

            
CIF Filename

            
Filename of the reference crystal structure file (CIF format). Must be placed in the same working directory as your Python script. Obtained from crystallographic databases such as ICSD, COD, or the Materials Project. Example: Mn2O3.cif

            
          


          
          

            
two_theta_rangeRequired

            
2θ Angular Range (min, max)

            
The angular window (in degrees) for generating the theoretical peak list (HKL file) from the CIF. This range should match or slightly exceed your measured data range. Reflections outside this window are excluded from the fit.

            

              
° min

              
° max

            

            
Typical: (10, 80) · must match measurement range

          


        


        
X-ray Source Wavelength

        

          ⚡ Select your X-ray source or enter custom wavelengths
          The wavelength list [λ₁, λ₂] specifies the primary (Kα₁) and secondary (Kα₂) emission lines of your X-ray source in Ångströms. For synchrotron sources, enter the monochromatic wavelength as both values (or set λ₂ = λ₁).

Primary emission wavelength of the X-ray source in Ångströms. Kα₁ is the dominant, shorter-wavelength component and contributes the majority of the diffraction signal intensity.

Secondary emission wavelength in Ångströms. Kα₂ is slightly longer and contributes ~50% of Kα₁ intensity, causing peak splitting at high angles. For monochromatic synchrotron sources, set equal to λ₁.

Step 4 — XRDfit Refinement Parameters
WPEM.XRDfit( wavelength, var, Lattice_constants, no_bac_file, original_file, bac_file, … )

        
      

      


        
Fitting Range & Angular Window

        


          
          

            
low_boundRequired

            
Lower 2θ Fitting Limit (°)

            
The minimum 2θ angle (degrees) included in the XRD profile fitting window. Peaks below this angle are excluded from the residual minimization. Should typically be set slightly above lowAngleRange used in background processing, to avoid background artefacts entering the fit.

            
°

            
Typical: 15 – 30° · must be > lowAngleRange

          


          
          

            
up_boundRequired

            
Upper 2θ Fitting Limit (°)

            
The maximum 2θ angle (degrees) for the fitting window. Should match or be slightly below the upper angular limit of your measured data. Higher angles provide more constraints on lattice parameters (d-spacing) but also amplify noise in laboratory diffractometers.

            
°

            
Typical: 60 – 120° · must be < two_theta_range max

          


        


        
WPEM Optimization Parameters

        


          
          

            
subset_numberOptional

            
WPEM Subset Count

            
Number of random sub-profiles sampled per EM iteration in the Expectation-Maximization algorithm. Larger values increase statistical robustness of the parameter update but require more memory and time per iteration. Equivalent to "mini-batch size" in machine learning terms.

            
            
Range: 1 – 200 · Typical: 5 – 20

          


          
          

            
iter_maxOptional

            
Maximum EM Iterations

            
Maximum number of outer EM (Expectation-Maximization) cycles. Each iteration updates the lattice constants and peak shape parameters. WPEM terminates early if convergence is reached. Increase for complex multi-phase systems or heavily distorted structures.

            
            
Range: 1 – 500 · Typical: 5 – 30

          


          
          

            
btaOptional

            
EM Convergence Rate (β)

            
Controls the learning rate of the EM parameter update — analogous to the momentum parameter in optimization. Values closer to 1.0 give slower but more stable convergence; values closer to 0.5 converge faster but may oscillate. For noisy data, use higher β (0.85–0.95).

            
            
Range: 0.1 – 1.0 · Typical: 0.75 – 0.95

          


          
          

            
asy_CAdvanced

            
Peak Asymmetry Coefficient

            
Correction coefficient for peak asymmetry caused by axial divergence of the diffractometer beam. Set to 0 for symmetric Voigt peaks (most cases). Positive values model low-angle asymmetry (peaks elongated toward lower 2θ). Relevant for high-resolution or high-count-rate measurements.

            
            
Range: −5 to 5 · Default: 0 (symmetric)

          


          
          

            
InitializationEpochAdvanced

            
Pre-Training Initialization Epochs

            
Number of neural-network pre-training epochs used to initialize the internal WPEM peak-shape model before the EM refinement begins. Setting this to 0 uses WPEM's built-in default Voigt initialization. Increase (e.g., 100–500) for complex peak shapes or when default initialization fails.

            
            
Range: 0 – 5000 · Default: 0
